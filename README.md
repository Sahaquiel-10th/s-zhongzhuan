# S 中转站

小规模、单实例部署的多租户 AI API 网关。客户只配置标准的 Base URL、API Key 和 Model；不再使用 S 中转站时，替换这三项配置即可，管理员只需停用旧 Key。

## 两种 API 服务模式

- **托管部署**：Key 绑定一个稳定的对外模型 ID，管理员控制实际上游模型和供应商。适合交付型项目。
- **开发者网关**：Key 绑定一个已开通的自助模型，使用方自主接入，管理员控制底层供应商线路。

同一账户可以同时拥有两种 Key，每枚 Key 只允许调用创建时选中的一个模型。管理员可在后台直接生成托管项目 Key，无需切换到使用页面。

每个供应商凭证需要选择 `OpenAI` 或 `Anthropic` 协议。网关透传各自的原生协议，不做跨协议翻译：OpenAI 使用 `/v1/chat/completions` 或 `/v1/responses`，Anthropic 使用 `/v1/messages`。选错端点会直接返回 `protocol_mismatch`，避免把不兼容请求发到上游。

绑定到单一 Anthropic 模型的 API Key 兼容 Claude Code 的 `fable`、`fable[1m]`、`sonnet`、`opus`、`opusplan`、`haiku` 和 `default` 等内置模型别名。别名只会路由到该 Key 已绑定的模型，并在转发前改写为供应商模型 ID，不会扩大 Key 的模型权限。

## 电力计费

数据库使用微电力作为最小单位：`1 电力 = 1,000,000 微电力`。每个 API 服务独立保存：

- 客户输入、缓存输入和输出价：参与实际扣费。
- 官方参考输入、缓存输入和输出价：只对客户展示。
- 供应商采购价：只对管理员展示，用于核算毛利。

```text
实扣电力
= 非缓存输入 Token ÷ 1,000,000 × 客户输入价
+ 缓存输入 Token ÷ 1,000,000 × 客户缓存输入价
+ 输出 Token ÷ 1,000,000 × 客户输出价
```

展示倍率由“客户价 ÷ 官方参考价”推导，不参与计费。输入和输出倍率可以不同。

## 价格发布与通知

管理员发布新价格时：

1. API 服务的价格版本自增。
2. 新价格对发布后的请求生效。
3. 系统生成一条持久通知。
4. 历史用量保留当时的价格、版本和标签快照。

使用控制台左下角的铃铛会在有新消息时显示红点，可查看并标记通知已读。API Key 也可读取：

```text
GET /v1/pricing
GET /v1/notices
```

模型响应包含 `X-S-Pricing-Version`、客户输入/缓存/输出价、官方参考价和 `X-S-Input-Factor` / `X-S-Output-Factor` 等响应头。非流式成功响应另包含 `X-S-Billed-Power`。

## 当前能力

- 管理员与普通账户两种登录角色；登录账号不要求是邮箱，旧邮箱账号保持兼容。
- 电力余额、API Key、服务、通知和调用记录按租户隔离。
- OpenAI `models`、`chat/completions`、`responses` 和 Anthropic `messages` 代理。
- 普通响应和 SSE 流式响应。
- 请求前预留电力，请求后按上游 usage 事务结算。
- API Key 使用单向哈希认证，并加密保存可恢复副本，页面保持遮盖但可再次复制；供应商 Key 使用 AES-256-GCM 加密。
- 调用日志和电力账本每页 10 条，支持按日期筛选并导出 CSV。
- 管理员后台和使用控制台分别提供面向新手的完整使用说明；管理员说明覆盖供应商 Key 录入、协议判断、模型上架、三组价格和交付 Key 流程。
- 微信支付 API v3 Native 扫码充值，支付成功后自动、幂等写入原电力账本；人工充值审核继续保留。
- 超管可以直接赠送电力，余额更新和账本记录在同一事务中完成；超管资金账本不展示逐笔 API 消耗。
- API Key 支持“先停用、再确认删除”，删除 Key 不会删除历史调用日志。
- 使用账号可验证旧密码后自行修改；管理员可重置临时密码，修改后旧登录会话自动失效。
- 供应商连接支持全平台多账户共享，可修改地址、协议、线路、启停和加密 Key；删除时会明确提示关联服务的影响。
- 微信充值支持固定套餐和自定义电力数量，最终金额始终由服务端按后台汇率计算。

## 微信 Native 支付

微信充值默认按 `1 电力 = 7 元人民币`，管理员可以在“计费说明”中修改。每笔订单保存创建时的汇率快照，后续修改只影响新订单。

回调地址固定为：

```text
https://你的正式域名/api/pay/wechat/notify
```

商户私钥和微信支付公钥放在服务器 `secrets/wechat/`，通过 Docker 只读挂载到 `/app/secrets/wechat/`。真实商户号、API v3 密钥和证书不得提交 Git；完整变量见 `.env.example`。

支付到账只认经过微信支付公钥验签、API v3 密钥解密并核对商户号、AppID、订单号、金额和币种的服务端回调。页面轮询只刷新状态，不能触发入账。

## 本地启动

```bash
cp .env.example .env
openssl rand -base64 32
# 把结果填入 UPSTREAM_KEY_ENCRYPTION_KEY，并修改管理员密码
npm install
npm run db:migrate
npm start
```

打开 `http://localhost:4173`。Docker 部署使用 `docker compose up --build -d`。

正式域名通过 `.env` 中的 `PUBLIC_BASE_URL=https://你的域名` 配置。部署脚本会保留服务器上已有的 `.env`，因此首次绑定域名或更换域名后，需要在服务器修改该值并重启服务。

## 数据库与备份

当前使用 SQLite WAL，适合小规模单实例部署。数据保存在 `data/super-relay.db`。应停止容器后备份整个 `data` 目录，或使用 SQLite 在线备份工具，不能只复制主数据库而忽略 WAL 文件。
